# 08. 最小 TUI

## 本步目标

做一个最小命令行界面，能看到模型流式输出和工具调用。

## 最小能力

| 能力 | 说明 |
| --- | --- |
| 输入任务 | 从命令行接收用户任务 |
| 流式文本 | 模型输出时实时显示 |
| 工具事件 | 显示工具名、参数和结果 |
| Todo 更新 | 把 `todo_write` 显示成任务列表 |
| 错误显示 | 请求或工具失败时显示错误 |

## 为什么先做命令行界面

TUI 比 Web UI 更容易验证 Runtime 是否正确。它不需要登录、路由、后端服务，也不需要浏览器状态。

## 当前运行方式

```bash
bun --env-file=.env.local packages/tui/src/cli.ts "读取 README 并总结项目结构"
```

也可以用根目录脚本：

```bash
bun run kairos "读取 README 并总结项目结构"
```

## 学到什么

界面层不要直接实现 Agent 逻辑。TUI 只负责输入、展示和错误呈现。

`todo_write` 是特殊工具：普通工具显示“开始/完成/失败”，它显示当前任务列表。这样用户能看到 Agent 的计划状态，而不是一段 JSON。
