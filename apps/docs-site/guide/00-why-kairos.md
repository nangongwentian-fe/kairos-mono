# 00. 为什么做 Kairos

## 本步目标

Kairos 的目标不是一次写出完整的 Agent 平台，而是按可验证的小步骤理解 Agent 的核心结构。

| 目标 | 含义 |
| --- | --- |
| 学会设计 | 知道每一层为什么存在，而不是只会调用现成框架 |
| 保持真实 | 尽早接入真实模型，暴露流式输出、工具调用、鉴权和错误 |
| 控制范围 | 每一步只做必要能力，避免一开始引入复杂系统 |
| 方便教学 | 每完成一步都能解释清楚，并能运行测试验证 |

## 我们不一开始做什么

| 暂不做 | 原因 |
| --- | --- |
| 多模型市场 | 先把一个真实 provider 做扎实 |
| 多 Agent 协作 | 单 Agent 的消息、工具、状态还没稳定 |
| 长期记忆 | 当前重点是运行时和工具安全 |
| 完整权限系统 | 先在 coding-agent 内做最小策略检查 |
| 图形化界面 | TUI 足够验证流式输出和工具事件 |

## 学到什么

Agent SDK 的学习顺序应该从“模型协议”和“运行循环”开始，再进入 coding tools、界面和任务管理。这样每一层的职责都能看清楚。
