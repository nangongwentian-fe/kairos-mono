# 00. 为什么做 Kairos

## 本步目标

Kairos 的目标是完整的 Agent 产品框架。实现节奏按可验证的小步骤推进，避免一次写太多导致 bug 增多，也降低 review 压力。

| 目标 | 含义 |
| --- | --- |
| 学会设计 | 知道每一层为什么存在，而不是只会调用现成框架 |
| 保持真实 | 尽早接入真实模型，暴露流式输出、工具调用、鉴权和错误 |
| 控制范围 | 每一步只做一个清楚的切片，但不否定长期产品边界 |
| 方便教学 | 每完成一步都能解释清楚，并能运行测试验证 |

## 实现节奏

| 做法 | 含义 |
| --- | --- |
| 先做基础边界 | 例如模型消息和 Agent 内部消息分层、事件、工具策略、会话状态 |
| 再做完整能力 | 例如长期记忆、上下文压缩、多 Agent、插件系统、远程运行环境 |
| 每次小切片 | 一个切片要能解释、能测试、能 review |
| 不照搬大系统 | 参考成熟项目的边界和取舍，不一次搬完整实现 |

## 学到什么

Agent SDK 的学习顺序应该从“模型协议”和“运行循环”开始，再进入 coding tools、界面和任务管理。这样每一层的职责都能看清楚。
