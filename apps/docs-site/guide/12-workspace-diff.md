# 12. Workspace Diff

## 本步目标

加入 workspace diff，让 Agent 和用户能看到当前工作区相对 git 的改动。

## 为什么需要 diff

| 场景 | 作用 |
| --- | --- |
| 完成任务前 | 看看改了哪些文件 |
| 测试失败后 | 判断是否引入了多余改动 |
| 用户教学 | 直观看到代码变化 |
| 后续提交 | 为提交说明提供依据 |

## 当前能力

| 能力 | 状态 |
| --- | --- |
| 读取 git diff | 支持 |
| 截断长输出 | 支持 |
| 非 git 仓库处理 | 支持错误返回 |
| 自动提交 | 暂不做 |

## 学到什么

coding agent 不只要会改代码，也要能解释自己改了什么。diff 是最直接的证据。
