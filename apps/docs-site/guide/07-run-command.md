# 07. 运行命令工具

## 本步目标

加入 `run_command`，让 Agent 可以运行测试、类型检查和项目命令。

## 最小设计

| 字段 | 作用 |
| --- | --- |
| `command` | 要运行的命令 |
| `cwd` | 工作目录，默认工作区 |
| `timeoutMs` | 超时时间 |
| `output` | 截断后的输出 |
| `exitCode` | 进程退出码 |

## 为什么要限制输出

命令输出可能很长。工具结果会进入模型上下文，所以要截断，避免一次失败日志占满上下文。

## 当前边界

| 能力 | 当前状态 |
| --- | --- |
| 运行 shell 命令 | 支持 |
| 超时控制 | 支持 |
| 输出截断 | 支持 |
| 复杂审批 | 暂不做 |
| 沙箱隔离 | 暂不做 |

## 学到什么

运行命令是 coding agent 进入真实开发流程的关键能力，但它比读文件和编辑文件风险更高，所以后续要配合工具策略。
