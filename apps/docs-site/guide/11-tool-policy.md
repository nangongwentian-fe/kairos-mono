# 11. Tool Policy

## 本步目标

在 `@kairos/coding-agent` 里加入工具策略，限制危险工具的使用方式。

## 为什么策略放在 coding-agent

`@kairos/agent` 不应该知道哪些命令危险，也不应该知道哪些路径能写。coding-agent 负责代码任务，所以策略放在这里更合适。

## 当前策略方向

| 工具 | 策略 |
| --- | --- |
| `read_file` | 限制工作区 |
| `list_dir` | 限制工作区 |
| `grep` | 限制工作区 |
| `edit_file` | 限制工作区，并要求先读 |
| `run_command` | 检查命令和工作目录 |

## 暂不做什么

| 暂不做 | 原因 |
| --- | --- |
| 完整权限系统 | 当前还没有多用户和远程执行场景 |
| 图形化审批 | TUI 还在最小阶段 |
| 复杂沙箱 | 先把策略接口稳定下来 |

## 学到什么

策略要靠近业务语义。通用 Runtime 提供挂载位置，coding-agent 决定具体规则。
