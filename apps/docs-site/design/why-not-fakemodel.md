# 为什么不先做 FakeModel

## 结论

FakeModel 要保留给测试，但不适合作为第一条学习主线。第一步接真实 OpenCode Go 更能帮助理解 Agent 的实际问题。

## 对比

| 维度 | FakeModel | OpenCode Go |
| --- | --- | --- |
| 流式输出 | 可以模拟 | 真实发生 |
| 工具调用分片 | 可以手写 | 需要按协议组装 |
| 鉴权错误 | 需要人为构造 | 会真实出现 |
| 模型行为差异 | 看不到 | 可以观察 |
| 测试稳定性 | 很高 | 适合做可选集成测试 |

## 当前策略

| 场景 | 使用方式 |
| --- | --- |
| 单元测试 | mock server 或测试工具 |
| 集成测试 | 有 `OPENCODE_API_KEY` 时可选运行 |
| 教学主线 | 真实 OpenCode Go |

## 学到什么

真实 provider 能让我们更早发现协议设计是否够用。FakeModel 后续可以作为测试基础设施补上。
