# 为什么是 grep 工具

## 结论

工具名选择 `grep`，不是泛化的 `search_text`。原因是 coding agent 用户通常理解 `grep` 的语义：在文件中搜索文本或正则。

## 对比

| 维度 | `search_text` | `grep` |
| --- | --- | --- |
| 名称直观性 | 偏泛化 | 接近开发者习惯 |
| 是否暗示正则 | 不明显 | 明确 |
| 是否容易扩展到 Web 搜索 | 容易混淆 | 不混淆 |
| 和 reference 项目一致性 | 较弱 | 更接近常见 coding agent |

## 当前工具能力

| 参数 | 作用 |
| --- | --- |
| `pattern` | 搜索文本或正则 |
| `path` | 搜索范围 |
| `include` | 文件匹配模式 |
| `maxResults` | 最大返回数量 |

## 学到什么

工具名本身也是提示。名字越贴近真实开发操作，模型越容易正确使用。
